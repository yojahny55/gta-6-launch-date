## Implementation Deep Dive

### Weighted Algorithm Specification

**Selected Approach: Hybrid Reasonable Range (Option C)**

**Weight Rules:**
- **Reasonable Range (2025-2030):** weight = 1.0 (full influence on median)
- **Outside but < 50 years:** weight = 0.3 (reduced influence)
- **Beyond 50 years:** weight = 0.1 (minimal influence, still counted and displayed)

**Calculation Method:**
1. Collect all predictions with their calculated weights
2. Sort predictions by date
3. Calculate weighted median: Find date where cumulative weight = 50% of total weight
4. Display: weighted median, actual min, actual max, total predictions

**User Transparency:**
- Disclaimer: "Extreme predictions have reduced influence on community median"
- Maintains democratic principle while protecting data integrity

### Technical Architecture

**Stack: Cloudflare Ecosystem (Free Tier)**
- **Frontend:** Cloudflare Pages (HTML/CSS/JS or React)
- **Backend:** Cloudflare Workers (API logic, 100k requests/day free)
- **Database:** Cloudflare D1 (SQLite, 5GB free)
- **Cache:** Cloudflare KV (optional, for performance)

**Database Schema:**
```sql
CREATE TABLE predictions (
  id INTEGER PRIMARY KEY,
  predicted_date DATE NOT NULL,
  submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  ip_hash TEXT NOT NULL,
  cookie_id TEXT NOT NULL,
  user_agent TEXT,
  weight REAL DEFAULT 1.0
);

CREATE TABLE email_subscriptions (
  id INTEGER PRIMARY KEY,
  email TEXT NOT NULL,
  cookie_id TEXT NOT NULL,
  subscribed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  verified BOOLEAN DEFAULT FALSE
);

CREATE INDEX idx_predictions_date ON predictions(predicted_date);
CREATE INDEX idx_predictions_cookie ON predictions(cookie_id);
```

**API Endpoints:**
- `POST /api/predict` - Submit new prediction with rate limiting
- `GET /api/stats` - Return aggregated statistics (cached)
- `PUT /api/predict/:cookie_id` - Update existing prediction
- `GET /api/embed` - Lightweight widget data
